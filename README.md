# Finger Chooser Web

스마트폰 브라우저에서 여러 사람이 손가락을 올리면 원이 생기고, 설정한 시간 동안 펄스/진동 효과가 나온 뒤 한 명을 랜덤으로 고르거나 순서를 정하는 웹 게임입니다.

이번 버전은 2가지 흐름을 지원합니다.

1. **빠른 손가락 선택**
   - 2~5명 정도가 한 번에 손가락을 올립니다.
   - 카운트가 끝나면 현재 손가락 중 1개를 고르거나, 현재 손가락들에 순서를 붙입니다.

2. **많은 사람 등록**
   - 6명 이상을 위해 만든 모드입니다.
   - 한 조 최대 5명씩 손가락을 올립니다. 기본값은 5명이며 설정에서 1~10명으로 바꿀 수 있습니다.
   - 카운트가 끝나면 각 손가락에 `번호 + 색 + 이모지 + 키워드` 이름표가 부여됩니다.
   - 등록 완료 후 손가락을 모두 떼면 다음 조를 등록할 수 있습니다.
   - 모든 조 등록이 끝나면 `등록자 실행` 버튼으로 전체 참가자 중 랜덤 1명을 뽑거나 전체 순서를 정합니다.

## 실행

```bash
cd finger_chooser_web
python3 server.py --host 0.0.0.0 --port 8000
```

같은 Wi-Fi에 연결된 스마트폰에서 서버가 출력하는 주소로 접속하세요.

예시:

```text
http://192.168.0.12:8000
```

## 사용법

### 빠른 손가락 선택

1. `방식`을 `빠른 손가락 선택`으로 둡니다.
2. `결과`에서 `랜덤 1명 선택` 또는 `순서 정하기`를 고릅니다.
3. 참가자들이 동시에 손가락을 올립니다.
4. 카운트가 끝날 때까지 계속 누르고 있습니다.
5. 결과가 화면에 표시됩니다.

### 많은 사람 등록

1. `방식`을 `많은 사람 등록`으로 바꿉니다.
2. 한 조 최대 인원을 확인합니다. 기본값은 5명입니다.
3. 첫 조가 손가락을 올리고 카운트가 끝날 때까지 유지합니다.
4. 등록된 사람은 아래 카드로 쌓입니다.
5. 등록 완료 후 손가락을 모두 뗀 뒤 다음 조를 등록합니다.
6. 모든 사람을 등록했으면 `등록자 실행`을 누릅니다.
7. `결과` 설정에 따라 전체 등록자 중 한 명이 선택되거나, 전체 순서가 부여됩니다.

## 버튼 설명

- `전체화면`: 브라우저 전체화면을 요청합니다.
- `등록자 실행`: 많은 사람 등록 모드에서 전체 등록자를 대상으로 결과를 냅니다.
- `현재 터치 초기화`: 현재 화면의 손가락/결과만 초기화합니다. 등록된 참가자 목록은 유지됩니다.
- `등록 전체 삭제`: 등록된 참가자 목록까지 모두 삭제합니다.

## iPhone에서 전체화면처럼 실행하기

Safari 안에서 `전체화면` 버튼을 누르면 iOS 정책상 차단될 수 있습니다. iPhone에서는 다음 방식이 가장 안정적입니다.

1. iPhone Safari에서 이 웹페이지를 엽니다.
2. 하단 또는 주소창의 공유 버튼을 누릅니다.
3. `홈 화면에 추가`를 선택합니다.
4. iOS 버전에 따라 `Open as Web App` 옵션이 보이면 켭니다.
5. 홈 화면에 생긴 Finger Chooser 아이콘으로 다시 실행합니다.

이 방식으로 열면 주소창과 하단 Safari UI가 줄어들어 앱처럼 사용할 수 있습니다.

## 구현 메모

- 모바일 실제 플레이는 `TouchEvent.touches`를 사용합니다.
- 데스크톱 테스트용으로 `PointerEvent` fallback도 넣었습니다.
- `navigator.maxTouchPoints`를 UI에 표시해 현재 기기의 이론상 최대 동시 터치 지원값을 확인합니다.
- `navigator.vibrate()`가 가능한 기기에서는 카운트다운과 결과 시점에 진동을 요청합니다. iOS Safari에서는 일반적으로 진동 API가 동작하지 않으므로 시각적 펄스 효과가 대체 피드백입니다.
- `중간에 손가락 수가 바뀌면 카운트 재시작` 옵션이 켜져 있으면 누군가 손가락을 떼거나 새로 올릴 때 카운트를 다시 시작합니다.
- iPhone 계열은 보통 5개 동시 터치를 안정 기준으로 잡는 것이 좋습니다. 그래서 많은 사람 등록 모드의 기본 한 조 최대값을 5명으로 설정했습니다.

## 2026-06-21 touch coordinate fix

- Touch/Pointer coordinates are now converted from viewport coordinates to the local `#stage` coordinate system using `stage.getBoundingClientRect()`.
- This fixes the issue where circles appeared lower than the actual finger position on mobile browsers.
- `index.html` adds a cache-busting query string to `app.js` so phones load the updated script.
